# AGENTS.md

This project is a TanStack Start portfolio site for Zoe Phiri, a web and brand designer. It is deployed on Netlify and uses Netlify Forms for contact submissions.

## Architecture

- `src/routes/__root.tsx` defines the HTML shell, document metadata, and global stylesheet import.
- `src/routes/index.tsx` contains the single-page portfolio experience, including navigation, services, portfolio examples, pricing, and the contact form.
- `src/styles.css` contains the full visual system: typography, layout, color tokens, responsive behavior, cards, buttons, and form styling.
- `public/contact.html` is a static hidden form skeleton. Netlify scans this file at build time so the React-rendered contact form can submit successfully.
- `netlify.toml` contains the Netlify build and publish configuration generated by the scaffold.

## Conventions

- Keep the site as a single-page experience unless a new route is explicitly requested.
- Use semantic sections with stable hash IDs for navigation.
- Use `lucide-react` icons for UI symbols.
- Preserve the Netlify Forms pattern: the React form in `src/routes/index.tsx` must keep field names aligned with `public/contact.html`, and AJAX submissions must post to `/contact.html`.
- Prefer edits in `src/routes/index.tsx` and `src/styles.css` for visual/content changes. Avoid reintroducing scaffold sample content unless the user asks for blog, resume, or project subpages.

## Non-Obvious Decisions

The original pasted HTML relied on external template SDK scripts and an in-browser data SDK. This project replaces those with a native React implementation and Netlify Forms so the enquiry workflow works in a normal Netlify deployment without depending on template-only runtime scripts.
